# ชุดข้อมูลการประเมิน

ชุดข้อมูล (Datasets) คือเป้าหมายคงที่ที่ harness ใช้ในการรัน แต่ละชุดข้อมูลเป็นไฟล์ JSON ที่ประกอบด้วยคู่ข้อมูล source→target พร้อมข้อมูลอ้างอิงมาตรฐานระดับทอง (gold-standard references) harness จะให้คะแนนผลลัพธ์ของโมเดลโดยเทียบกับข้อมูลอ้างอิงเหล่านี้ — และจะไม่มีการแก้ไขข้อมูลอ้างอิงโดยเด็ดขาด

:::danger ห้ามใช้ข้อมูลการประเมินในการฝึกสอน

⚠️ **ชุดข้อมูลเหล่านี้ใช้สำหรับการประเมินเท่านั้น** วิธีการใดๆ ที่ผ่านการฝึกสอน (trained), ปรับแต่งแบบละเอียด (fine-tuned), ใช้ few-shot-prompted หรือสัมผัสกับข้อมูลการประเมินในทางใดทางหนึ่ง จะทำให้ได้คะแนนสูงเกินจริงและจะถูก **ตัดสิทธิ์จากกระดานผู้นำ (leaderboard)**

โปรดใช้คลังข้อมูล (corpora) แยกต่างหากสำหรับการฝึกสอน ชุดข้อมูลการประเมินจะต้องไม่ถูกโมเดลของคุณมองเห็นในระหว่างการพัฒนา
:::

## รูปแบบชุดข้อมูล

ทุกชุดข้อมูลจะใช้โครงสร้าง JSON (JSON schema) รูปแบบเดียวกัน:

```json
{
  "dataset": {
    "id": "dataset-slug",
    "version": "1.0",
    "language_pair": "EN→CRK",
    "description": "Human-readable description of the dataset",
    "source_language": "en",
    "target_language": "crk",
    "created": "2025-05-01",
    "license": "CC-BY-NC-4.0",
    "provenance": ["gold_standard", "textbook"]
  },
  "entries": [
    {
      "index": 0,
      "source_text": "Hello",
      "target_expected": "tânisi",
      "difficulty": "easy",
      "provenance": "gold_standard",
      "notes": "Common greeting, SRO orthography"
    }
  ]
}
```

### บล็อก `dataset` ระดับบนสุด

| ฟิลด์ | ประเภท | คำอธิบาย |
|-------|------|-------------|
| `id` | `string` | ตัวระบุชุดข้อมูลที่ไม่ซ้ำกัน (ใช้ใน run cards และ leaderboard) |
| `version` | `string` | Semantic version การเพิ่มเวอร์ชันนี้จะทำให้การเปรียบเทียบ run card ก่อนหน้าเป็นโมฆะ |
| `language_pair` | `string` | ป้ายกำกับสำหรับแสดงผล (เช่น `EN→CRK`) |
| `description` | `string` | บทสรุปที่มนุษย์สามารถอ่านได้ |
| `source_language` | `string` | รหัสภาษาต้นทางตามมาตรฐาน BCP 47 |
| `target_language` | `string` | รหัสภาษาปลายทางตามมาตรฐาน BCP 47 |
| `created` | `string` | วันที่สร้างตามมาตรฐาน ISO 8601 |
| `license` | `string` | ตัวระบุใบอนุญาตตามมาตรฐาน SPDX |
| `provenance` | `string[]` | รายการแท็กแหล่งที่มา (provenance tags) ที่ใช้ในรายการข้อมูลทั้งหมด |

### ฟิลด์รายการข้อมูล (Entry Fields)

| ฟิลด์ | ประเภท | คำอธิบาย |
|-------|------|-------------|
| `index` | `number` | ดัชนีรายการที่เริ่มต้นด้วยศูนย์ (Zero-based) ต้องไม่ซ้ำกันและเรียงตามลำดับ |
| `source_text` | `string` | ข้อความต้นทางที่ต้องการแปล |
| `target_expected` | `string` | คำแปลอ้างอิงมาตรฐานระดับทอง (gold-standard reference) |
| `difficulty` | `string` | ระดับความยาก: `easy`, `medium`, `hard` |
| `provenance` | `string` | แหล่งที่มาของรายการนี้ (เช่น `gold_standard`, `textbook`, `elicited`) |
| `notes` | `string` | บริบทเพิ่มเติม (ไม่บังคับ) สำหรับผู้ตรวจสอบที่เป็นมนุษย์ |

## ชุดข้อมูลที่มีให้ใช้งาน

### EDTeKLA Development Set v1

ชุดข้อมูลการประเมินชุดแรก ซึ่งสร้างขึ้นสำหรับการแปลภาษา English→Plains Cree (SRO) สร้างโดย [EdTeKLA research group](https://spaces.facsci.ualberta.ca/edtekla/) ที่ University of Alberta

| คุณสมบัติ | ค่า |
|----------|-------|
| **ID** | `edtekla-dev-v1` |
| **Version** | `1.0` |
| **คู่ภาษา** | EN → CRK (Plains Cree, ระบบการเขียน SRO) |
| **จำนวนรายการ** | 124 |
| **การกระจายระดับความยาก** | ง่าย (Easy), ปานกลาง (Medium), ยาก (Hard) |
| **แหล่งที่มา** | `gold_standard` (ตรวจสอบโดยผู้พูดจริง), `textbook` (สื่อการสอนที่ได้รับการตีพิมพ์) |
| **ใบอนุญาต** | [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/) |

**สิ่งที่ใช้ทดสอบ:**

- คำทักทายพื้นฐานและวลีทั่วไป
- Noun animacy และ obviation
- การผันคำกริยาตามบุรุษและกาล (tenses)
- โครงสร้างบอกสถานที่ (Locative constructions)
- กระบวนทัศน์การแสดงความเป็นเจ้าของ (Possessive paradigms)
- โครงสร้างประโยคที่ซับซ้อน

:::tip ทำไมถึงมี 124 รายการ?
ชุดข้อมูลนี้จงใจทำให้มีขนาดเล็กและผ่านการคัดสรรมาอย่างดี แต่ละรายการได้รับการตรวจสอบโดยผู้พูดที่ใช้ภาษาได้อย่างคล่องแคล่ว หรือนำมาจากหนังสือเรียนภาษา Cree ที่ได้รับการตีพิมพ์ ชุดข้อมูลขนาดเล็กที่มีคุณภาพสูงพร้อมข้อมูลอ้างอิงมาตรฐานระดับทองที่ผ่านการตรวจสอบแล้วนั้น มีประโยชน์มากกว่าชุดข้อมูลขนาดใหญ่ที่มีข้อมูลรบกวน (noisy) — โดยเฉพาะอย่างยิ่งสำหรับภาษาที่มีทรัพยากรน้อย (low-resource language) ซึ่งคำแปลที่ "ใกล้เคียง" มักจะไม่ถูกต้องตามหลักสัณฐานวิทยา (morphologically invalid)
:::

## การสร้างชุดข้อมูลใหม่

หากคุณต้องการสร้างชุดข้อมูลสำหรับคู่ภาษาหรือโดเมนใหม่:

### 1. กำหนดโครงสร้าง JSON

ปฏิบัติตามโครงสร้าง [รูปแบบชุดข้อมูล](#dataset-format) ทุกรายการจะต้องมี `source_text`, `target_expected`, `difficulty`, และ `provenance`

### 2. กำหนด ID ที่ไม่ซ้ำกัน

ใช้ slug ที่สื่อความหมาย: `{project}-{split}-v{version}` (เช่น `edtekla-dev-v1`, `quechua-test-v1`)

### 3. ตรวจสอบมาตรฐานระดับทอง (Gold standards)

ทุกค่าของ `target_expected` จะต้องได้รับการตรวจสอบโดยผู้พูดที่ใช้ภาษาได้อย่างคล่องแคล่ว หรือนำมาจากแหล่งข้อมูลที่ได้รับการตีพิมพ์และผ่านการตรวจสอบโดยผู้เชี่ยวชาญ (peer-reviewed) ข้อมูลอ้างอิงที่สร้างด้วยเครื่อง (Machine-generated) จะทำให้วัตถุประสงค์ของการประเมินล้มเหลว

### 4. กำหนดระดับความยาก

กำหนดระดับความยากให้กับแต่ละรายการ:

| ระดับ | เกณฑ์ |
|------|----------|
| `easy` | วลีสั้นๆ, คำศัพท์ทั่วไป, สัณฐานวิทยาแบบง่าย |
| `medium` | ประโยคสมบูรณ์, มีความซับซ้อนทางสัณฐานวิทยาระดับปานกลาง |
| `hard` | ไวยากรณ์ซับซ้อน, โครงสร้างที่พบได้ยาก, เนื้อหาที่มีความเฉพาะเจาะจงทางวัฒนธรรม |

### 5. แท็กแหล่งที่มา (Provenance)

แต่ละรายการควรระบุว่ามาจากที่ใด แท็กที่พบบ่อยได้แก่:

- `gold_standard` — ตรวจสอบโดยผู้พูดที่ใช้ภาษาได้อย่างคล่องแคล่ว
- `textbook` — จากสื่อการสอนที่ได้รับการตีพิมพ์
- `elicited` — สร้างขึ้นผ่านเซสชันการดึงข้อมูลแบบมีโครงสร้าง (structured elicitation sessions)
- `corpus` — สกัดมาจากคลังข้อมูลคู่ขนาน (parallel corpus)

### 6. ตรวจสอบความถูกต้องของไฟล์

รัน harness กับชุดข้อมูลของคุณด้วยโมเดลใดก็ได้ เพื่อตรวจสอบว่า JSON มีรูปแบบที่ถูกต้องและมีฟิลด์ที่จำเป็นครบถ้วน:

```bash
python eval/baseline_experiment.py --dataset path/to/your-dataset.json
```

harness จะแสดงข้อผิดพลาดหากมีฟิลด์หายไป, ดัชนีซ้ำกัน, หรือมีการละเมิดโครงสร้าง (schema violations)

### 7. ส่งข้อมูลเพื่อขอเพิ่มเข้าระบบ

เปิด pull request ไปที่ [eval harness repository](https://github.com/gamedaysuits/gds-mt-eval-harness) พร้อมแนบไฟล์ชุดข้อมูลของคุณในไดเรกทอรี `data/` โปรดรวมเอกสารประกอบเกี่ยวกับระเบียบวิธีในการตรวจสอบและแหล่งที่มาของคุณด้วย

## FLORES+ Devtest

เกณฑ์มาตรฐาน (benchmark) แบบหลายภาษาที่ครอบคลุมในวงกว้าง ดูแลโดย [Open Language Data Initiative (OLDI)](https://huggingface.co/datasets/openlanguagedata/flores_plus) ใช้สำหรับ frontier benchmark แบบหลายโมเดลของ rosetta

| คุณสมบัติ | ค่า |
|----------|-------|
| **ID** | `flores-plus-devtest` |
| **คู่ภาษา** | EN → 39 ภาษา (ภาษาธรรมชาติทั้งหมดที่ลงทะเบียนใน rosetta) |
| **จำนวนรายการ** | 1,012 ประโยคต่อภาษา |
| **ใบอนุญาต** | [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/) |
| **แหล่งที่มา** | เดิมคือ Meta FLORES-200 ปัจจุบันดูแลโดย OLDI |
| **ตำแหน่งที่ตั้ง** | Fixtures ที่สกัดไว้ล่วงหน้าอยู่ที่ `test/benchmark/fixtures/` ใน repo หลักของ rosetta |

:::danger สำหรับการประเมินเท่านั้น
FLORES+ มีไว้สำหรับการประเมินเท่านั้น ผู้ดูแลระบบขอความร่วมมืออย่างชัดเจนว่า **ห้ามใช้เป็นข้อมูลการฝึกสอน** โปรดตรวจสอบให้แน่ใจว่าเนื้อหาของชุดข้อมูลนี้ถูกแยกออกจากคลังข้อมูลการฝึกสอนใดๆ
:::
