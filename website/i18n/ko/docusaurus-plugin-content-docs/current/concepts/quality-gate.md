# Quality Gate

모든 번역은 디스크에 기록되기 전에 결정론적 유효성 검사 게이트를 통과합니다. Quality Gate는 기계 번역의 일반적인 실패 유형을 포착하여, 알림 없이 대체(silent fallback)되거나 로캘 파일에 불필요한 데이터가 기록되는 것을 방지합니다.

## 유효성 검사

| 검사 항목 | 포착 내용 | 게이트 레이블 |
|-------|----------------|-----------|
| **빈 문자열/공백** | 모델이 빈 문자열이나 공백을 반환함 | `[GATE] empty` |
| **원본 에코** | 모델이 원본 영어 입력을 그대로 반환함 | `[GATE] source-echo` |
| **환각 루프** | 반복되는 트라이그램(trigram) 패턴 (예: `"Qo' Qo' Qo'"`) | `[GATE] hallucination` |
| **길이 팽창** | 출력 결과가 원본보다 지나치게 긺 | `[GATE] length` |
| **문자 체계 준수** | 대상 로캘에 맞지 않는 문자 체계 | `[GATE] script` |

### 빈 문자열/공백

빈 문자열, 공백으로만 이루어진 문자열 또는 `null`인 번역을 거부합니다. 이는 번역하기 어려운 키에 대해 모델이 아무것도 반환하지 않는 경우를 포착합니다.

### 원본 에코

모델이 번역을 수행하지 않고 영어 원문을 그대로 반환하는 경우를 감지합니다. 짧은 문자열이나 프롬프트가 불충분하게 지정된 경우에 흔히 발생합니다.

### 환각 루프

출력 결과에서 트라이그램(3글자) 패턴을 분석합니다. 출력 길이에 비례하여 임계값 이상으로 반복되는 트라이그램이 있는 경우 해당 번역은 거부됩니다. 이는 `"Qo' Qo' Qo' Qo' Qo'"`와 같이 변질된 출력 결과를 포착합니다.

### 길이 팽창

출력 길이가 `maxLengthRatio × source length`(기본값: 4배)를 초과하는 번역을 거부합니다. 이는 짧은 입력에 대해 장문의 텍스트를 생성하는 모델의 환각(hallucination) 현상을 포착합니다.

구성 파일의 `maxLengthRatio`을 통해 설정할 수 있습니다.

### 문자 체계 준수

`script` 필드가 구성된 로캘(예: 평원 크리어 음절 문자의 경우 `"script": "cans"`)에 대해, 출력 결과에 대상 문자 체계에 적합한 비 ASCII 문자가 포함되어 있는지 유효성을 검사합니다. 아랍어, CJK(한중일) 또는 음절 문자 로캘에 대해 라틴 문자로만 구성된 출력은 거부됩니다.

## 실패 시 동작

1. 실패한 번역은 `[GATE]` 접두사, 키 이름, 실패 사유 및 값의 미리보기와 함께 stderr에 기록됩니다.
2. 해당 키는 로캘 파일에 기록되지 **않습니다**.
3. 재시도 캐스케이드(retry cascade)가 시작됩니다(아래 참조).

```
[GATE] hero.title: source-echo — "Welcome to our platform"
[GATE] nav.about: hallucination — "À À À À À À À À"
```

## 재시도 캐스케이드

배치(batch)가 실패하면(JSON 구문 분석 오류 또는 Quality Gate 거부), rosetta는 점진적으로 더 작은 배치 크기로 재시도합니다.

```
Full batch (30 keys) → parse error
  └→ Half batch (15 keys) → 2 failures
      └→ Individual keys (1 each) → isolates the 2 problem keys
```

재시도 예산은 `maxRetries`에 의해 제한됩니다(기본값: 3, 언어별로 설정 가능). 이를 통해 지속적으로 실패하는 키에 대해 토큰이 과도하게 소비되는 것을 방지합니다.

재시도를 모두 소진하면 문제의 키가 기록되고 건너뜁니다. 해당 키들은 다음 `sync` 실행 시 다시 시도됩니다.

## 프롬프트 캐싱

시스템 메시지(어조, 문법 규칙, 스타일 참고 사항)는 사용자 메시지(번역할 키)와 분리됩니다. 이러한 분리는 의도적입니다.

- 특정 로캘에 대해 시스템 메시지는 **모든 배치에서 동일**합니다.
- Anthropic 및 Google과 같은 제공업체는 반복되는 시스템 메시지를 캐시합니다.
- 결과: 첫 번째 배치는 전체 토큰 비용을 지불하지만, 이후의 배치들은 사용자 메시지에 대한 비용만 지불합니다.

이를 통해 배치가 많은 프로젝트의 토큰 비용을 크게 절감할 수 있습니다.
