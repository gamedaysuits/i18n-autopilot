# MT評価

rosettaには、翻訳メソッドの**再現可能なベンチマーク**のために設計された機械翻訳評価フレームワークが含まれています。これは特に、標準的なMTベンチマークが存在せず、品質の主張を検証することが困難な、リソースの少ない言語や先住民言語のために設計されています。

---

## リーダーボード

その中心となるのが**[メソッドリーダーボード](/leaderboard)**です。これはSupabaseをバックエンドとしたライブのスコアボードであり、研究者やコミュニティのメンバーが、フィンガープリント付きの再現可能な評価とともに翻訳メソッドを提出し、比較することができます。

すべての提出物には以下が含まれます：

- **フィンガープリント付きパイプライン** — 特定のGitコミットと設定ハッシュに結び付けられているため、結果を生成した正確なコードまで遡ることができます。
- **バージョン管理されたデータセット** — コンテンツがハッシュ化され、バージョン管理されています。スコアは同じデータセットのバージョン内でのみ比較可能です。
- **標準化されたメトリクス** — すべてのスコアリングは共有の評価ハーネスによって計算され、実装の違いを排除します。
- **信頼性ティア** — 自己ベンチマーク（Self-benchmarked）、GDS検証済み（GDS Verified）、またはコミュニティ検証済み（Community Validated）
- **コスト追跡** — 提出ごとのAPIコスト。これにより、コストと品質のトレードオフが透明になります。

リーダーボードでは現在、3つのメトリクスを追跡しています：

| メトリクス | タイプ | 測定内容 |
|--------|------|------------------|
| **chrF++** | 文字n-gram F値 | 主要な品質メトリクス — 特に形態論的に豊かな言語において、人間の判断とよく相関します。 |
| **Exact Match** | 完全一致の割合 | 厳密な精度 — 翻訳がゴールドスタンダードと完全に一致する頻度。 |
| **FST Acceptance** | 形態素ゲート通過率 | 有限状態トランスデューサ（FST）検証を伴うメソッド向け — 出力のどの程度の割合が形態論的に有効か。 |

**[→ リーダーボードを見る](/leaderboard)**

## 利用可能なデータセット

### EDTeKLA Development Set v1

英語から平原クリー語（SRO）への翻訳のために構築された、最初の評価データセットです。アルバータ大学の[EdTeKLA研究グループ](https://spaces.facsci.ualberta.ca/edtekla/)によって作成されました。

| プロパティ | 値 |
|----------|-------|
| **ID** | `edtekla-dev-v1` |
| **言語ペア** | EN → CRK（平原クリー語、SRO正書法） |
| **エントリ数** | 124 |
| **ライセンス** | [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/) |
| **出所** | `gold_standard`（話者による検証済み）、`textbook`（出版された教育資料） |

### FLORES+ Devtest

[Open Language Data Initiative (OLDI)](https://huggingface.co/datasets/openlanguagedata/flores_plus)によって維持されている、広範囲をカバーする多言語ベンチマークです。

| プロパティ | 値 |
|----------|-------|
| **言語ペア** | EN → 39言語（rosettaに登録されているすべての言語） |
| **エントリ数** | 各言語1,012文 |
| **ライセンス** | [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/) |
| **ソース** | 元はMeta FLORES-200、現在はOLDIが維持 |
| **場所** | rosettaのメインリポジトリ内の`test/benchmark/fixtures/`にある事前抽出されたフィクスチャ |

完全なデータセットスキーマ、難易度ティア、および独自のデータセットの作成方法については、[評価データセット](/docs/eval/datasets)を参照してください。

:::danger 評価データでトレーニングしないこと

**これらのデータセットは評価専用です。** 評価データでトレーニング、ファインチューニング、Few-shotプロンプトの実行、またはその他の方法で評価データに触れたメソッドは、人為的に水増しされたスコアを生成するため、**リーダーボードから失格となります。**

これは提案ではありません。評価の完全性における最も重要な単一のルールです。トレーニングには別のコーパスを使用してください。評価セットは、開発中にモデルから見えない状態を保つ必要があります。

コーチングデータやFew-shotの例を使用する場合、それらは**完全に別のソース**からのものである必要があります。疑わしい場合は、含めないでください。
:::

:::warning LLMの非決定性

LLMの出力は非決定的です。スコアは、特定のモデルバージョンとAPI設定の下でのある時点の測定値を表します。モデルプロバイダーは、重み、デコード戦略、またはセーフティフィルターをいつでも更新する可能性があり、これにより実行間でスコアの変動が生じる可能性があります。リーダーボードには、すべての提出物について正確なモデルスラッグとタイムスタンプが記録されます。
:::

## 優れたメソッドの条件

すべてのメソッドが平等に作られているわけではありません。厳密な作業と水増しされたスコアを分けるものは以下の通りです。

### 強力なメソッドの特徴

- **トレーニングデータと評価データの明確な分離** — 開発、チューニング、プロンプトエンジニアリング、またはFew-shotの例の選択中に、メソッドが評価セットを一度も見たことがないこと。
- **再現性** — 他の誰かがリポジトリをクローンし、ハーネスを実行して、（LLMの非決定性の範囲内で）同じスコアを取得できること。
- **文書化** — [メソッドカード](/docs/eval/methods)に、メソッドの機能、使用するツール、および制限事項が記載されていること。
- **スコープに対する誠実さ** — メソッドが1つの言語ペアでしか機能しない場合は、その旨を記載すること。特定の形態論的パターンで劣化する場合は、それを文書化すること。
- **コミュニティへの配慮** — 先住民言語の場合、メソッドがデータ主権を尊重していること。言語コミュニティと協議したか、オープンライセンスのデータのみを使用していること。

### 危険信号（失格になるもの）

| 危険信号 | 問題となる理由 |
|----------|--------------------|
| 評価データでのトレーニング | 評価の目的を完全に無意味にします。水増しされたスコアはすべての人を誤解させます。 |
| 結果のチェリーピッキング | 10回実行し、他の結果を開示せずに最高の結果のみを提出すること。 |
| 未開示の後処理 | スコアリングの前に出力を手動で修正すること。 |
| 汚染されたコーチングデータ | 評価セットの例をFew-shotプロンプトや辞書のエントリとして使用すること。 |
| 出所不明な商用利用可能の主張 | メソッドがCC BY-NC-SAデータを使用している場合、商用利用の準備はできていません。 |

### リーダーボードの品質ティア

リーダーボードは3つの信頼性レベルをサポートしています：

| ティア | 意味 | 取得方法 |
|------|---------|---------------|
| **自己ベンチマーク (Self-benchmarked)** | 自分でハーネスを実行し、結果を提出した | ランカード（run card）を添付してPRを開く |
| **GDS検証済み (GDS Verified)** | rosettaのメンテナーが結果を再現した | インストール可能なプラグインとしてメソッドを提出する |
| **コミュニティ検証済み (Community Validated)** | 独立したコミュニティメンバーが結果を再現した | 近日公開 |

## 提出方法

1. **メソッドを構築する** — メソッドのインターフェースについては、[メソッドの構築](/docs/eval/methods)を参照してください。
2. **ハーネスを実行する** — セットアップと使用方法については、[評価ハーネス](/docs/eval/harness)を参照してください。
3. **ランカードを生成する** — ハーネスは、スコア、フィンガープリント、メタデータを含むJSONランカードを生成します。
4. **PRを開く** — ランカードを[評価ハーネスのリポジトリ](https://github.com/gamedaysuits/gds-mt-eval-harness)に提出します。
5. **リーダーボードに掲載される** — マージされると、結果が[メソッドリーダーボード](/leaderboard)に表示されます。

## 今後の方向性

- **FLORES+モデル比較の実行** — すべての39のrosetta言語にわたる、フロンティアモデル（GPT-5.5、Claude Opus 4.7、Gemini 3.1 Proなど）の体系的な評価。
- **より多くの言語ペア** — コミュニティで検証されたデータセットが利用可能になり次第、ケチュア語、イヌクティトゥット語、およびその他のリソースの少ない言語を追加。
- **データセットのインポート** — 外部の評価データセット（WMT、Tatoebaなど）をrosettaの評価フォーマットに変換するためのツール。
- **自動再実行** — モデルのバージョン変更を検出し、ベンチマークを再実行してスコアの変動を追跡。
