# i18n-rosetta

完全にカスタマイズ可能な国際化（i18n）フレームワークです。1つのコマンドでロケールファイルを翻訳します。1つの設定ファイルで、すべてのメソッド、モデル、言語ペアを制御します。組み込みのメソッドで不十分な場合は、独自のメソッドを構築し、その動作を証明してデプロイすることができます。

```bash
npx i18n-rosetta sync
```

rosettaは、ロケールファイル、フォーマット、ターゲット言語を自動検出します。不足しているものを翻訳し、完了しているものをスキップし、すべての結果を検証して、クリーンな出力を書き出します。これがスタートラインです。

---

## なぜ自分でスクリプトを書かないのか？

各キーに対してGoogle Translateを呼び出す簡単なループを書くことは可能です。多くの開発者がそうしており、約30行のコードで済みます。しかし、以下の点で破綻します：

- **変更検出がない。** 英語の文字列を更新しても、翻訳は古いまま永遠に残ります。rosettaは、すべてのソース値をSHA-256ハッシュで追跡し、変更されたものだけを再翻訳します。
- **バッチ処理がない。** 1キーにつき1回のAPI呼び出しを行うと、200キーで200回の通信が発生します。rosettaはインテリジェントにバッチ処理を行います（設定可能、デフォルトはLLMで30キー/バッチ、Googleで128キー/バッチ）。
- **キャッシュがない。** 同期するたびにすべてを再翻訳します。rosettaのTranslation Memoryは、ソーステキスト＋ロケール＋メソッドごとに翻訳をキャッシュします。1つのキーを変更した後に同期を再実行しても、ファイル全体ではなく、その1つのキーだけが翻訳されます。
- **品質ゲートがない。** 機械翻訳はハルシネーションを起こしたり、ソースをそのまま返したり、間違った文字体系で出力したりします。rosettaは書き込む前にすべての翻訳を検証し、文字体系の誤り、不自然な長さの増加、ソースのそのままの出力などを検出して拒否します。
- **フォーマット認識がない。** JSONにハードコードされていませんか？rosettaは、JSON、TOML、YAML、Hugo Markdown（フロントマター＋本文）を自動検出して処理します。
- **メソッド制御がない。** すべての言語ペアで同じメソッドが使用されます。rosettaを使用すると、フランス語にはGoogle Translate、日本語にはLLM、クリー語にはコミュニティがホストするカスタムパイプラインを、同じ設定ファイル内で使用できます。

rosettaは、そのスクリプトのプロダクション版です。

## 何が違うのか

### すべてのメソッドがプラグイン

翻訳メソッドは**言語ペアごとに設定可能**です。Google Translate、LLM、コーチングされたプロンプト、カスタムAPIを同じプロジェクト内で組み合わせることができます：

```json title="i18n-rosetta.config.json"
{
  "version": 3,
  "pairs": {
    "en:fr": { "method": "google-translate" },
    "en:ja": { "method": "llm", "model": "google/gemini-2.5-pro" },
    "en:crk": { "methodPlugin": "crk-coached-v1" }
  }
}
```

フランス語にはGoogle Translate（高速、安価）を使用します。日本語にはプレミアムなLLM（ニュアンスを考慮）を使用します。平原クリー語には、文法規則、辞書、形態素検証を備えたコーチング済みプラグインを使用します。同じ `sync` コマンド、同じ品質ゲート、同じCLIを使用します。

### 証明する

あなたのメソッドが英語からスペイン語へ、トルコ語からアゼルバイジャン語へ、英語からクリー語へ翻訳できると思いますか？

**それを証明してください。** 付属の[評価ハーネス](https://mtevalarena.org/docs/specifications/harness)は、再現性のあるフィンガープリント付きのスコアリングで、あらゆる翻訳メソッドをベンチマークします。[リーダーボード](/leaderboard)はすべての提出を追跡します。

評価ハーネスとプロダクションCLIは、同じプラグインインターフェースを共有しています。ハーネスで高スコアを獲得したメソッドは、その言語を使用するコミュニティが同意すれば、本番環境で使用できます。先住民の言語やリソースの少ない言語にとって、その同意は重要です。[データ主権](https://mtevalarena.org/docs/sovereignty/data-sovereignty)をご覧ください。

```bash
# Benchmark your method (in the eval harness repo)
cd gds-mt-eval-harness
python eval/baseline_experiment.py --dataset data/edtekla-dev-v1.json --submit

# Use it locally
npx i18n-rosetta sync
```

同じプラグインです。組み込んでテストしてください。

### 完全なツールキット

rosettaは単なる `sync` ではありません。完全なi18nパイプラインです：

| コマンド | 機能 |
|---------|-------------|
| `sync` | 不足しているキー、古いキー、フォールバックキーを翻訳します |
| `watch` | ソースファイルが変更されたときに自動同期します |
| `lint` | ソースコードをスキャンしてハードコードされた文字列を探します |
| `wrap` | ハードコードされた文字列を `t()` 呼び出しで自動ラップします |
| `audit` | 翻訳されていない `[EN]` フォールバック値をすべてリストアップします |
| `integrity` | プレースホルダーの破損、エンコーディングの問題、ICUの複数形の完全性を検出します |
| `seo` | hreflangタグ、サイトマップ、JSON-LDスキーマを生成します |
| `status` | ペア設定、プラグイン、ベンチマークスコアを表示します |
| `provenance` | 翻訳リソースのライセンスを監査します |
| `plugin` | メソッドプラグインのインストール、削除、リスト表示を行います |
| `fonts` | PUA文字体系コンバーター用のWebフォントをダウンロードします |
| `tm` | Translation Memoryキャッシュを管理します（統計、クリア、ロケールごと） |
| `xliff` | プロの翻訳者によるレビュー用にXLIFF 1.2をエクスポート/インポートします |

これらのうち3つ（`lint`、`sync`、`audit`）は、ハードコードされた文字列を検出し、それらを翻訳し、不完全なロケールがある場合はビルドを失敗させるCIパイプラインを形成します。

## アリーナ

[メソッドリーダーボード](/leaderboard)はスコアボードです。すべての提出物はGitコミットにフィンガープリントされ、特定のデータセットにバージョン付けされ、同じハーネスによってスコアリングされます。誰でも提出できます。

**何を証明できますか？** ハーネスはJSONを受け取ります。プラグインもJSONを受け取ります。JSONを出力するメソッドであれば、何でもテストできます：

| アプローチ | 例 |
|----------|---------|
| **コーチング済みLLM** | フロンティアモデルのプロンプトに文法規則と辞書を注入します |
| **ファインチューニング済みモデル** | パラレルテキストでオープンモデルをトレーニングします（ただし評価データは除外） |
| **FSTゲートパイプライン** | LLMが生成 → 有限状態トランスデューサー（FST）が形態素を検証 → 再試行 |
| **連鎖モデル** | モデルAが下書き → モデルBがポストエディット → モデルCがスコアリング |
| **辞書 + LLM** | 辞書から既知の用語を強制し、残りをLLMに処理させます |
| **進化的アルゴリズム** | 候補を生成し、スコアリングし、最良のものを変異させ、繰り返します |
| **部分翻訳** | サンプルを手動で翻訳し、LLMが一致することを証明し、残りを自動翻訳します |

モデルをファインチューニングしたり、進化的アルゴリズムをデプロイしたり、語学試験の学生の回答をテストしたり、ルックアップテーブルを構築したり、3つのモデルを連鎖させたりすることができます。メソッドがJSONを出力する限り、ハーネスはそれをスコアリングし、フレームワークはそれを実行します。

:::danger 唯一のルール
**評価データでトレーニングしないでください。** ベンチマークデータセットにさらされたメソッドは失格となります。好きなものでファインチューニングしてください。ただし、テストセットは除外してください。
:::

これはオープンな招待状です。研究者、コミュニティメンバー、学生、あるいは単に関心がある人として、リソースの少ない言語に取り組んでいるのであれば、メソッドを構築し、ハーネスを実行して、トップスコアを獲得してください。問題は未解決です。インフラはここにあります。
