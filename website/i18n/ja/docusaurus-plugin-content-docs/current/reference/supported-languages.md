# 対応言語

rosettaには、50言語に対応した構造化設定ファイルである**Language Cards**が同梱されています。各カードには、レジスター（使用域）のプリセット、フォーマル度のメタデータ、メソッドのサポートフラグ、タイポグラフィのルール、および文字情報が含まれています。LLMが認識している言語であれば、設定を1行追加するだけで任意の言語を追加できます。ここに挙げているのは、本番環境ですぐに使えるよう調整されたレジスターを持つ言語です。

---

## 翻訳メソッド

各言語では、以下の翻訳メソッドを1つ以上使用できます。

| アイコン | メソッド | 仕組み | コスト |
|------|--------|-------------|------|
| 🟢 | **Google Translate** | ニューラル機械翻訳のベースライン。130以上の言語に対応。キーと値の文字列のみ対応 — Markdownコンテンツを安全に翻訳することはできません。 | 約$20/100万文字 |
| 🔵 | **LLM (OpenRouter)** | モデルが認識しているすべての言語に対応。レジスター制御のプロンプト。キーと値の文字列およびMarkdownコンテンツを処理します。 | モデルにより異なる |
| 🟣 | **LLM-Coached** | LLM + 文法辞書 + プロンプトに注入されるコーチングデータ。形態論的に複雑な言語に最適です。 | モデルにより異なる |
| 🟠 | **API (Plugin)** | HTTP経由で提供される、コミュニティホスト型の翻訳パイプライン。[OCAP互換](https://mtevalarena.org/docs/community/low-resource-languages)。 | プロバイダーにより異なる |

Google Translateの場合は`GOOGLE_TRANSLATE_API_KEY`を、LLMメソッドの場合は`OPENROUTER_API_KEY`を設定します。詳細については、[翻訳メソッド](/docs/guides/translation-methods)を参照してください。

## 先住民言語および低資源言語

これらの言語は、商用の機械翻訳サービスではサポートされていません。rosettaは、言語コミュニティが[OCAP原則](https://mtevalarena.org/docs/community/low-resource-languages)に基づいて独自のメソッドを構築するためのツールを提供します。

| | 言語 | コード | Google | LLM | Coached | 文字 | ステータス |
|---|----------|------|:------:|:---:|:-------:|--------|--------|
| 🪶 | 平原クリー語 | `crk` | ❌ | ✅ | ✅ | 🔤 SRO→音節文字 | 🚧 開発中 |
| 🌄 | ケチュア語 | `qu` | ✅ | ✅ | — | — | Runasimi。証拠性の接尾辞。 |

:::info 平原クリー語は現在開発中です
平原クリー語のレジスター、コーチングインフラ、文字コンバーター、および評価ハーネスはすべて機能していますが、翻訳パイプラインは**まだリリースされていません**。リリース前に品質を確保するため、[OCAP原則](https://mtevalarena.org/docs/community/low-resource-languages)に基づいて言語コミュニティと協力しています。詳細および貢献方法については、[低資源言語のサポート](https://mtevalarena.org/docs/community/low-resource-languages)を参照してください。
:::

:::tip さらに低資源言語を追加する
rosettaのメソッドプラグインシステムは、このために設計されています。言語コミュニティはカスタムの翻訳メソッドを構築し、独自の管理下でホストして、[APIメソッド](/docs/guides/serving-a-method)経由で提供することができます。[メソッドリーダーボード](/leaderboard)では、あらゆる言語ペアのスコアを追跡しています。メソッドを構築し、ハーネスを実行して、トップスコアを獲得しましょう。
:::

## 人工言語

人工言語（Conlangs）は、LLMレジスターとオプションの文字コンバーターを通じてサポートされています。これらは実際の言語と同じインフラを使用しており、品質ゲート、コーチングシステム、および文字変換パイプラインはまったく同じように機能します。

| | 言語 | コード | Google | LLM | 文字 | 備考 |
|---|----------|------|:------:|:---:|--------|-------|
| 🖖 | クリンゴン語 | `tlh` | ❌ | ✅ | 🔤 ローマ字→pIqaD | PUAフォントが必要。マーク・オークランドの語彙。 |
| 🧝 | シンダリン (トールキンのエルフ語) | `x-elvish-s` | ❌ | ✅ | 🔤 ラテン文字→テングワール | CSUR PUAフォントが必要。 |
| 🏴‍☠️ | 海賊英語 | `x-pirate` | ❌ | ✅ | — | レジスターのみ。航海のメタファー。 |
| 🦸 | クリプトン語 | `x-kryptonian` | ❌ | ✅ | 🔤 ラテン文字→クリプトン文字 | PUAフォントが必要。 |
| 🎭 | シェイクスピア英語 | `x-shakespeare` | ❌ | ✅ | — | レジスターのみ。Thee/thou、-eth/-estフォーム。 |
| 🐸 | ヨーダ語 | `x-yoda` | ❌ | ✅ | — | レジスターのみ。OSVの語順。 |

PUAフォントの要件、Unicodeの制限、および独自の言語を追加する方法については、[人工言語、文字、正書法](/docs/guides/conlangs-scripts-orthography)を参照してください。

## 言語プリセット

`init`ウィザードは、迅速なセットアップのためのプリセット名をサポートしています。プリセットと個別のコードを混在させることができます。

| プリセット | 展開後 |
|--------|-----------|
| `european` | fr, de, es, it, pt, nl |
| `asian` | ja, zh, ko |
| `global` | fr, es, de, ja, zh, ko, pt, ar |
| `nordic` | da, fi, nb, sv |

```bash
# Mix presets with individual codes
i18n-rosetta init
# → Target languages: european, ja
# → Resolves to: fr, de, es, it, pt, nl, ja
```

## 任意の言語の追加

rosettaは、**LLMが認識している任意の言語**に翻訳できます。上の表は、組み込みのレジスタープリセットを持つ言語をリストしているにすぎません。リストにない言語を追加するには、設定にそのBCP-47コードを含めます。

```json
{
  "languages": {
    "sw": {},
    "am": {
      "register": "Formal Amharic. Professional register with Geʽez script."
    }
  }
}
```

LLMは、その言語に関する学習知識を使用して翻訳します。`register`を設定することで、トーン、フォーマル度、および正書法の規則を制御できます。詳細については、[設定](/docs/getting-started/configuration)を参照してください。

## Language Cards

組み込みの各言語には**Language Card**があります。これは、パフォーマンスのために2つの階層に分割された構造化JSON設定です。

### 2層アーキテクチャ

| 階層 | ディレクトリ | 読み込みタイミング | 目的 |
|------|-----------|--------|--------|
| **Runtime** | `lib/data/language-cards/` | `import`時に即時読み込み | 翻訳エンジン: レジスター、フォーマル度、ルール、メソッドサポート |
| **Reference** | `lib/data/language-reference/` | 必要に応じて遅延読み込み | 開発者向けドキュメント: 言語的な課題、百科事典データ、NLPリソース |

rosettaをインポートする際に数メガバイトのドキュメントデータが読み込まれないよう、Runtime階層は小さく（カードあたり約2 KB）保たれています。Reference階層は、ツール、Webサイト、および評価ハーネス向けに`getLanguageReference(code)`経由で利用できます。

### Runtimeカードのフィールド

| フィールド | 内容 |
|-------|------------------|
| **`nativeName`** | エンドニム — その言語自身の文字による自称（例: ქართული、Runasimi） |
| **Formality system** | T-V区分、スピーチレベル、敬語、助詞など |
| **Register presets** | その言語の特性に合わせた名前付きLLMプロンプトのプリセット |
| **Method support** | どの翻訳APIがこの言語をサポートしているか |
| **Gender guidance** | 文法上の性別のルールとインクルーシブなライティングのヒント |
| **Script/direction** | ISO 15924文字コードとRTL/LTR |
| **Rules** | タイポグラフィ（引用符、スペース）、大文字化、複数形のカテゴリ |
| **Eval datasets** | どのベンチマークがこの言語をカバーしているか |
| **`glottocode`** | 相互参照用の正規Glottolog識別子 |
| **`humanReviewed`** | カードが話者によってレビューされたかどうか |

### Referenceカードのフィールド

| フィールド | 内容 |
|-------|------------------|
| **Linguistic challenges** | 機械翻訳特有の落とし穴（例: 証拠性、声調記号、膠着性） |
| **Encyclopedic** | 語族、分類、話者数、地域 |
| **Resources** | NLPツール、対訳コーパス、事前学習済みモデル |

### 新しいLanguage Cardのひな形作成

ジェネレーターを使用して、信頼できるデータソース（IANA、CLDR、Glottolog）から両方の階層のひな形を作成します。

```bash
# Preview what would be generated
node scripts/generate-language-card.mjs sw --dry-run

# Generate both runtime + reference cards
node scripts/generate-language-card.mjs sw
```

ジェネレーターはメタデータ（コード、文字、方向、複数形、引用符、メソッドサポート、語族）を自動入力し、言語的な判断が必要なフィールドを人間のキュレーション用にTODOとしてマークします。

### プリセットキーの使用

完全なレジスターのテキストを記述する代わりに、プリセットキー名を使用できます。

```json
{
  "languages": {
    "fr": "casual-tu",
    "ko": "formal-hapsyo",
    "ja": "polite"
  }
}
```

rosettaは、キーを完全なレジスタープロンプトに解決します。各言語で利用可能なプリセットを確認するには、`npx i18n-rosetta init`を実行してください。

### プリセットの例

| 言語 | プリセット | デフォルト |
|----------|---------|--------|
| フランス語 | `formal-vous`, `casual-tu` | `formal-vous` |
| 韓国語 | `polite-haeyo`, `formal-hapsyo`, `casual-hae` | `polite-haeyo` |
| 日本語 | `polite`, `formal-keigo`, `casual` | `polite` |
| ドイツ語 | `formal-Sie`, `casual-du` | `formal-Sie` |
| タイ語 | `neutral-professional`, `polite-male`, `polite-female` | `neutral-professional` |
| スペイン語 | `neutral-professional`, `formal-usted`, `casual-tuteo` | `neutral-professional` |

フィールドの検証やPRチェックリストを含む完全な仕様については、[Language Cardのコントリビュート](https://github.com/gamedaysuits/i18n-rosetta)を参照してください。
