# セキュリティと安全性

Rosettaは、ロケールデータが信頼できないソースから提供される場合や、細工されたファイル名がディレクトリの境界を越えようとする場合、LLMの出力に何が含まれるか分からない場合など、敵対的な環境でも安全に機能するように設計されています。

## 脅威モデル

| 脅威 | 攻撃ベクター | 緩和策 |
|--------|--------------|-----------|
| **プロトタイプ汚染** | 細工されたJSONキー (`__proto__`, `constructor`) | パース時に拒否 |
| **パストラバーサル** | `../../etc/passwd` のようなロケールコード | ファイル書き込みを設定されたディレクトリに検証 |
| **コードブロックの破損** | LLMがコードフェンス内を翻訳する | Unicodeセンチネルによる保護 |
| **ハルシネーションによるキー** | LLMが送信されていないキーを返す | レスポンスの検証 — 許可されたキーのみを書き込み |
| **トークン消費の暴走** | 無限リトライループ | `maxRetries` による予算上限設定 |

## プロトタイプ汚染ガード

すべてのロケールキーは、処理前にブロックリストと照合して検証されます。

- `__proto__`
- `constructor`
- `prototype`

これらのパターンに一致するキーはエラーとして拒否されます。これにより、攻撃者が細工されたロケールファイルを使用してJavaScriptオブジェクトのプロトタイプを変更するのを防ぎます。

## パスの制限

ロケールファイルを書き込む際、rosettaは出力パスが設定されたディレクトリ (`localesDir`, `contentDir`) 内に収まっていることを検証します。ロケールコードはサニタイズされるため、`../../secrets` のようなコードが想定外のディレクトリに書き込まれることはありません。

## ブロック保護

Markdownコンテンツの翻訳中、構造化された要素は、テキストがLLMに送信される前にUnicodeセンチネルのプレースホルダーに置き換えられます。

1. **コードブロック** (フェンスおよびインライン) → センチネル
2. **Hugoショートコード** (`{{< >}}`, `{{% %}}`) → センチネル  
3. **生のHTML** → センチネル
4. **補間変数** (`{{ .Count }}`) → センチネル

翻訳後、センチネルは元のコンテンツに置き換えられます。LLMがコードブロック、ショートコード、HTMLを見ることはないため、これらが破損することはありません。

## レスポンスの検証

LLMがJSONレスポンスを返した際、rosettaは以下のことを検証します。
- バッチで送信されたキーのみがレスポンスに含まれていること
- 余分なキーが注入されていないこと
- レスポンスが有効なJSONとしてパースできること

ハルシネーションによるキーは静かに破棄されます。これにより、LLMの出力がロケールファイルに予期しない翻訳を注入するのを防ぎます。

## 品質ゲート

すべての翻訳は、ディスクに書き込まれる前に5つの決定論的なチェックを通じて検証されます。詳細については、[品質ゲート](/docs/concepts/quality-gate)を参照してください。

## 指数バックオフ

API呼び出しは、429 (レート制限) および 5xx (サーバーエラー) のレスポンスに対して、ジッター付きの指数バックオフを使用します。遅延を増やしながら3回のリトライを行うことで、障害発生時にAPIに過剰な負荷をかけるのを防ぎます。

## リクエストタイムアウト

すべてのAPIリクエストには、`AbortController` による30秒のタイムアウトが設定されています。これにより、切断された接続で同期プロセスが無限にハングするのを防ぎます。

## フォールバックモード

APIが利用できない場合、`--fallback` は実際の翻訳の代わりに `[EN]` のプレフィックスが付いたプレースホルダーを書き込みます。

```bash
npx i18n-rosetta sync --fallback
```

```json
{
  "hero.title": "[EN] Welcome to our platform"
}
```

これらのプレースホルダーは自動的に検出され、有効なAPIキーを使用した次回の同期時に再翻訳されます。これらが「翻訳済み」として扱われることはなく、`audit` がフラグを立てます。

## テスト

セキュリティの特性は、敵対的テストスイートによって検証されます。

```bash
npm run test:redteam    # prototype pollution, path traversal, encoding attacks
```
