# 安装

## 环境要求

- **Node.js 20.11+** (原生 ESM)
- 翻译 API 密钥（见下文）

## 安装

<Tabs>
  <TabItem value="npm" label="npm" default>

```bash
npm install --save-dev i18n-rosetta
```

  </TabItem>
  <TabItem value="npx" label="npx (no install)">

```bash
npx i18n-rosetta sync
```

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

```bash
pnpm add -D i18n-rosetta
```

  </TabItem>
  <TabItem value="yarn" label="yarn">

```bash
yarn add --dev i18n-rosetta
```

  </TabItem>
</Tabs>

## 获取 API 密钥

Rosetta 需要一个翻译后端。请选择其中一个：

| 提供商 | 环境变量 | 最佳适用场景 |
|----------|---------------------|----------|
| **OpenRouter**（推荐） | `OPENROUTER_API_KEY` | 内容密集的项目、Markdown、200+ 模型 |
| **Google Translate** | `GOOGLE_TRANSLATE_API_KEY` | 大量键值对（130+ 语言） |

<Tabs>
  <TabItem value="openrouter" label="OpenRouter (recommended)" default>

[OpenRouter](https://openrouter.ai) 为 200 多种 LLM 模型提供统一的 API。提供免费额度。

```bash
# Sign up at https://openrouter.ai, then:
export OPENROUTER_API_KEY=sk-or-v1-...
```

或者添加到 `.env.local` 中：

```bash
OPENROUTER_API_KEY=sk-or-v1-your-key-here
```

最佳适用场景：内容密集的项目、Markdown 翻译，以及需要内容感知屏蔽（代码块、短代码、插值变量）的项目。

  </TabItem>
  <TabItem value="google" label="Google Translate">

```bash
export GOOGLE_TRANSLATE_API_KEY=...
```

最佳适用场景：大量键值对字符串（130+ 语言）。**不推荐**用于 Markdown 内容 —— Google Translate 无法识别代码块、短代码或插值变量。

:::tip 智能检测
如果仅设置了 `GOOGLE_TRANSLATE_API_KEY`（没有 OpenRouter 密钥），rosetta 会自动切换到 Google Translate。无需更改配置。
:::

  </TabItem>
</Tabs>

## 验证安装

```bash
npx i18n-rosetta --help
```

你应该能看到可用命令的列表。你现在可以进入[快速开始](/docs/getting-started/quick-start)了。
