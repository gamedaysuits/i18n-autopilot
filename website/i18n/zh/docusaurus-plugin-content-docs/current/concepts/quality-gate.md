# 质量门禁

每次翻译在写入磁盘前都会经过确定性的验证门禁。质量门禁可捕获常见的机器翻译故障模式——没有静默回退，也不会向您的本地化文件写入垃圾数据。

## 验证检查

| 检查项 | 捕获内容 | 门禁标签 |
|-------|----------------|-----------|
| **空值/空白** | 模型返回空字符串或空格 | `[GATE] empty` |
| **源文回显** | 模型返回原始英文输入 | `[GATE] source-echo` |
| **幻觉循环** | 重复的三元组模式（例如，`"Qo' Qo' Qo'"`） | `[GATE] hallucination` |
| **长度膨胀** | 输出明显长于源文 | `[GATE] length` |
| **字符集规范** | 目标语言区域的字符集错误 | `[GATE] script` |

### 空值/空白

拒绝空字符串、仅含空格或 `null` 的翻译。这可捕获模型在处理复杂键时未返回任何内容的情况。

### 源文回显

检测模型是否返回英文源文而未进行翻译。这在短字符串和提示词说明不足时很常见。

### 幻觉循环

分析输出中的三元组（3 字符）模式。如果任何三元组的重复次数相对于输出长度超过了阈值，则拒绝该翻译。这可捕获如 `"Qo' Qo' Qo' Qo' Qo'"` 类的退化输出。

### 长度膨胀

拒绝输出长度超过 `maxLengthRatio × source length`（默认：4 倍）的翻译。这可捕获模型因幻觉而对简短输入生成大段文本的情况。

可通过配置中的 `maxLengthRatio` 进行配置。

### 字符集规范

对于配置了 `script` 字段的语言区域（例如，平原克里语音节文字的 `"script": "cans"`），验证输出是否包含适合目标字符集的非 ASCII 字符。如果阿拉伯语、CJK（中日韩）或音节文字语言区域的输出仅包含拉丁字母，则会被拒绝。

## 失败时的处理机制

1. 失败的翻译将记录到 stderr，并带有 `[GATE]` 前缀、键名、原因以及值的预览
2. 该键**不会**写入本地化文件
3. 触发重试级联（见下文）

```
[GATE] hero.title: source-echo — "Welcome to our platform"
[GATE] nav.about: hallucination — "À À À À À À À À"
```

## 重试级联

当批处理失败（JSON 解析错误或被质量门禁拒绝）时，rosetta 会使用逐渐减小的批次进行重试：

```
Full batch (30 keys) → parse error
  └→ Half batch (15 keys) → 2 failures
      └→ Individual keys (1 each) → isolates the 2 problem keys
```

重试预算上限由 `maxRetries` 决定（默认：3，可按语言配置）。这可防止在持续失败的键上出现 token 消耗失控的情况。

耗尽重试次数后，问题键将被记录并跳过。它们将在下一次 `sync` 运行时重试。

## 提示词缓存

系统消息（语域、语法规则、样式说明）与用户消息（要翻译的键）是分开的。这种分离是有意为之的：

- 对于特定的语言区域，系统消息在**所有批次中完全相同**
- Anthropic 和 Google 等提供商会缓存重复的系统消息
- 结果：第一个批次支付全额 token 成本，后续批次仅需为用户消息支付成本

这可以显著降低包含大量批次项目的 token 成本。
