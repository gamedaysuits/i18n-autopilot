الإصدار v3.2.0 هو إصدار الجودة. 702 اختبار، 163 حزمة اختبار، ولا تسامح مطلقاً مع الإخفاقات الصامتة.

## ما الذي تغير

### Quality Gate (5 فحوصات)

تمر كل ترجمة الآن عبر خمسة فحوصات تحقق حتمية قبل كتابتها على القرص:

1. **فارغ/خالٍ** — لم يُرجع النموذج أي شيء
2. **تكرار المصدر** — أرجع النموذج المدخلات الإنجليزية
3. **حلقة الهلوسة** — أنماط trigram متكررة
4. **تضخم الطول** — المخرجات أطول بـ 4 أضعاف أو أكثر من المصدر
5. **الامتثال للنظام الكتابي** — نظام كتابة خاطئ للغة المحلية

لا تُكتب أي ترجمة دون اجتياز الفحوصات الخمسة جميعها. يتم تسجيل الترجمات الفاشلة وإعادة المحاولة.

### تسلسل إعادة المحاولة

عندما تفشل دفعة (batch)، يعيد rosetta المحاولة بدفعات أصغر تدريجياً:

```
Full batch (30 keys) → parse error
  └→ Half batch (15 keys) → 2 failures
      └→ Individual keys (1 each) → isolates the problem keys
```

### تعزيز الأمان

- **حماية Prototype pollution** — يتم رفض المفاتيح `__proto__` و `constructor` وقت التحليل
- **حماية Path traversal** — لا يمكن لرموز اللغات المحلية المصممة التلاعب والكتابة خارج الدلائل التي تم تكوينها
- **التحقق من الاستجابة** — يتم قبول المفاتيح التي تم إرسالها فقط عند إرجاعها

### Prompt Caching

يتم الآن فصل رسائل النظام عن رسائل المستخدم، مما يتيح الاستفادة من Prompt Caching لدى مزودين مثل Anthropic و Google. يقلل هذا بشكل كبير من تكاليف الـ token لعمليات المزامنة متعددة الدفعات.
