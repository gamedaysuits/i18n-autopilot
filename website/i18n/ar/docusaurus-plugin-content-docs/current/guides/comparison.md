# مقارنة Rosetta

تحتل i18n-rosetta فئة مختلفة عن معظم أدوات التوطين. إليك مقارنة صريحة.

## المشهد العام

تندرج معظم أدوات التوطين تحت واحدة من ثلاث فئات:

| الفئة | أمثلة | النموذج |
|----------|----------|-------|
| **منصات TMS السحابية** | Crowdin, Phrase, Locize, Tolgee | لوحة تحكم SaaS + مترجمون بشريون + اشتراك شهري |
| **أدوات استخراج المفاتيح** | i18next-scanner, FormatJS CLI | فحص الكود المصدري بحثًا عن استدعاءات دوال الترجمة |
| **محركات ترجمة واجهة سطر الأوامر (CLI)** | **i18n-rosetta** | تعمل داخل مشروعك، تترجم الملفات مباشرة، بدون حساب سحابي |

Rosetta هي **محرك ترجمة يعمل عبر واجهة سطر الأوامر (CLI)** — فهي تترجم ملفات اللغات الخاصة بك مباشرة باستخدام خلفيات قابلة للتهيئة (LLMs، Google Translate، إضافات مخصصة). لا توجد لوحة تحكم سحابية، ولا سير عمل للمترجمين البشريين، ولا رسوم شهرية.

## متى تستخدم Rosetta

**تُعد Rosetta خيارًا مناسبًا عندما:**

- ترغب في دمج الترجمة الآلية ضمن مسار البناء (build pipeline) الخاص بك — وليس كسير عمل منفصل
- تحتاج إلى التحكم في طريقة الترجمة لكل لغة على حدة (LLM للبعض، Google Translate للبعض الآخر، وإضافات مخصصة للبقية)
- تترجم إلى لغات لا تدعمها واجهات برمجة التطبيقات (API) (مثل اللغات الأصلية، المهددة بالانقراض، أو المصطنعة)
- ترغب في الحصول على مخرجات نصية حتمية (مثل Cree Syllabics، Klingon pIqaD، Tengwar)
- لا تريد أي تقييد بمُورّد (vendor lock-in) أو أي تبعيات سحابية
- كنت مطورًا مستقلاً أو فريقًا صغيرًا لا يحتاج إلى سير عمل للمترجمين البشريين

**تُعد منصات TMS السحابية خيارًا أفضل عندما:**

- يكون لديك مترجمون بشريون محترفون يراجعون كل نص
- تحتاج إلى ذاكرة ترجمة وإدارة للمصطلحات عبر المشاريع
- تحتاج إلى تحرير مرئي في السياق (معاينة الترجمات داخل واجهة المستخدم الخاصة بك)
- يكون لديك فريق كبير يحتاج إلى تحكم في الوصول بناءً على الأدوار (role-based access control)
- تحتاج إلى دعم لأكثر من 50 صيغة ملفات

## ما تقدمه Rosetta ولا يقدمه غيرها

### 1. مستويات لغة (Registers) مخصصة

يحصل كل زوج لغوي على تعليمات نبرة مناسبة ثقافيًا لنموذج اللغة الكبير (LLM):

```json
{
  "de": {
    "register": "Standard professional register. Use Sie-form for formal address."
  },
  "tl": {
    "register": "Educated Manila Taglish. Use Tagalog as the primary language but keep technical terms in English."
  },
  "tlh": {
    "register": "Warrior's honor. OVS grammar. Use Marc Okrand vocabulary."
  }
}
```

لا توجد أداة أخرى تأتي مزودة بـ 47 مستوى لغويًا مُعدًا مسبقًا، أو تتيح لك تحديد مستويات مخصصة لكل مشروع.

### 2. محولات نصوص حتمية (Deterministic Script Converters)

تأتي Rosetta مزودة بخمسة محولات نصوص مدمجة تعمل كخطافات (hooks) بعد الترجمة — دون الحاجة إلى LLM:

| اللغة (Locale) | التحويل | مثال |
|--------|-----------|---------|
| `crk` | SRO → Cree Syllabics | `nêhiyawêwin` → `ᓀᐦᐃᔭᐍᐏᐣ` |
| `sr` | Latin → Cyrillic | `Beograd` → `Београд` |
| `tlh` | Romanization → pIqaD | `tlhIngan Hol` → (رموز pIqaD) |
| `x-elvish-s` | Latin → Tengwar | Sindarin → Tengwar (وضع Beleriand) |
| `x-kryptonian` | Latin → Kryptonian | استبدال التشفير (يتطلب خطًا) |

هذه محولات تعتمد بالكامل على جداول البحث (lookup-table) — فهي حتمية، وقابلة للتدقيق، وبنسبة صفر من مخاطر هلوسة LLM.

### 3. حماية تدرك المحتوى (Content-Aware Shielding)

عند ترجمة Markdown أو المحتوى الغني، تقوم Rosetta بحماية:

- كتل الأكواد المحاطة (` ``` `)
- الأكواد المضمنة (Inline code) (`` ` ` ``)
- الأكواد القصيرة لـ Hugo (`{{</* */>}}`, `{{%/* */%}}`)
- متغيرات الاستيفاء (Interpolation variables) (`{{ .Count }}`, `{name}`, `{{t('key')}}`)
- كتل HTML الخام

يتم استبدال هذه العناصر برموز حراسة (sentinel tokens) من Unicode قبل الترجمة واستعادتها بعد ذلك. لا يرى LLM أبدًا الكود الخاص بك، أو أكوادك القصيرة، أو متغيراتك.

### 4. إضافات طرق الترجمة الموجهة (Coached Method Plugins)

بالنسبة للغات التي لا تغطيها واجهات برمجة التطبيقات (API)، يمكنك بناء طريقة ترجمة موجهة:

1. كتابة بيانات التوجيه اللغوي (القواعد النحوية، المفردات، الأمثلة)
2. حزمها كإضافة (plugin)
3. قياس أدائها مقابل الترجمات المرجعية باستخدام [eval harness](https://github.com/gamedaysuits/gds-mt-eval-harness)
4. تثبيتها في مشروعك باستخدام `i18n-rosetta plugin install`

هذه هي الطريقة التي تتعامل بها rosetta مع لغة Plains Cree — وكيف يمكنك التعامل مع أي لغة، بما في ذلك اللغات التي لم توجد بعد.

## الخلاصة

لا تُعد Rosetta بديلاً عن Crowdin. إنها أداة مختلفة لسير عمل مختلف. إذا كنت بحاجة إلى مترجمين بشريين، فاستخدم منصة TMS. أما إذا كنت بحاجة إلى واجهة سطر أوامر (CLI) تترجم ملفاتك بأمر واحد وتمنحك تحكمًا لكل لغة في الطرق والنماذج ومستويات اللغة — فاستخدم rosetta.
