# i18n-rosetta

إطار عمل للتدويل (internationalization) قابل للتخصيص بالكامل. أمر واحد يترجم ملفات اللغات الخاصة بك. إعداد واحد يتحكم في كل طريقة ونموذج وزوج لغوي. وإذا لم تكن الطرق المدمجة كافية — قم ببناء طريقتك الخاصة، وأثبت فعاليتها، ثم انشرها.

```bash
npx i18n-rosetta sync
```

تكتشف rosetta تلقائيًا ملفات اللغات الخاصة بك وتنسيقها واللغات المستهدفة. تترجم ما هو مفقود، وتتخطى ما تم إنجازه، وتتحقق من صحة كل نتيجة، وتكتب مخرجات نظيفة. هذه هي نقطة البداية.

---

## لماذا لا تكتب نصاً برمجياً بنفسك؟

يمكنك كتابة حلقة تكرار (loop) سريعة تستدعي Google Translate لكل مفتاح. يفعل معظم المطورين ذلك — يستغرق الأمر حوالي 30 سطرًا. ولكن هنا تفشل الطريقة:

- **عدم اكتشاف التغييرات.** عند تحديث نص إنجليزي — تظل الترجمة قديمة إلى الأبد. تتتبع rosetta كل قيمة مصدرية باستخدام تجزئات SHA-256 (hashes) وتعيد ترجمة ما تغير فقط.
- **عدم تجميع الطلبات (Batching).** إجراء استدعاء API واحد لكل مفتاح يعني 200 مفتاح = 200 طلب ذهاب وإياب. تقوم rosetta بتجميع الطلبات بذكاء (قابل للإعداد، الافتراضي 30 مفتاحًا/دفعة لـ LLM، و 128 لـ Google).
- **عدم وجود تخزين مؤقت (Caching).** كل عملية مزامنة تعيد ترجمة كل شيء. تقوم ذاكرة الترجمة (Translation Memory) في rosetta بتخزين الترجمات مؤقتًا بناءً على النص المصدري + اللغة + الطريقة — إعادة تشغيل المزامنة بعد تغيير مفتاح واحد تترجم هذا المفتاح فقط، وليس الملف بأكمله.
- **عدم وجود بوابة جودة.** الترجمة الآلية قد تهلوس، أو تعيد النص المصدري كما هو، أو تخرج نصًا بنظام كتابة خاطئ. تتحقق rosetta من كل ترجمة قبل كتابتها — حيث يتم اكتشاف ورفض أنظمة الكتابة الخاطئة، والتضخم في طول النص، وتكرار النص المصدري.
- **عدم التعرف على التنسيقات.** هل الكود مقتصر على JSON؟ تتعامل rosetta مع JSON و TOML و YAML و Hugo Markdown (البيانات الوصفية + المحتوى) مع الاكتشاف التلقائي.
- **عدم التحكم في الطريقة.** يحصل كل زوج لغوي على نفس الطريقة. تتيح لك rosetta استخدام Google Translate للغة الفرنسية، و LLM للغة اليابانية، ومسار مخصص مستضاف من قبل المجتمع للغة Cree — في نفس ملف الإعدادات.

rosetta هي نسخة الإنتاج (production) من ذلك النص البرمجي.

## ما الذي يجعلها مختلفة

### كل طريقة هي إضافة (Plugin)

طريقة الترجمة **قابلة للإعداد لكل زوج لغوي**. يمكنك الدمج بين Google Translate، ونماذج LLM، والموجهات الموجهة (coached prompts)، وواجهات برمجة التطبيقات (APIs) المخصصة في نفس المشروع:

```json title="i18n-rosetta.config.json"
{
  "version": 3,
  "pairs": {
    "en:fr": { "method": "google-translate" },
    "en:ja": { "method": "llm", "model": "google/gemini-2.5-pro" },
    "en:crk": { "methodPlugin": "crk-coached-v1" }
  }
}
```

تحصل الفرنسية على Google Translate (سريع ورخيص). وتحصل اليابانية على LLM متميز (دقيق). وتحصل لغة Plains Cree على إضافة موجهة تحتوي على قواعد نحوية وقواميس وتحقق صرفي. نفس الأمر `sync`. نفس بوابة الجودة. نفس واجهة سطر الأوامر (CLI).

### أثبت ذلك

هل تعتقد أن طريقتك يمكنها الترجمة من الإنجليزية إلى الإسبانية؟ أو من التركية إلى الأذربيجانية؟ أو من الإنجليزية إلى Cree؟

**أثبت ذلك.** تقوم [أداة التقييم](https://mtevalarena.org/docs/specifications/harness) المرافقة بقياس أداء أي طريقة ترجمة بنتائج قابلة للتكرار وموثقة ببصمة. وتتتبع [لوحة المتصدرين](/leaderboard) كل إرسال.

تشترك أداة التقييم وواجهة سطر الأوامر (CLI) الخاصة بالإنتاج في نفس واجهة الإضافات (plugin interface). الطريقة التي تسجل نتائج جيدة في التقييم يمكن استخدامها في الإنتاج — إذا وافق المجتمع الذي تخدمه لغته على ذلك. بالنسبة للغات الأصلية واللغات ذات الموارد المحدودة، هذه الموافقة مهمة. راجع [سيادة البيانات](https://mtevalarena.org/docs/sovereignty/data-sovereignty).

```bash
# Benchmark your method (in the eval harness repo)
cd gds-mt-eval-harness
python eval/baseline_experiment.py --dataset data/edtekla-dev-v1.json --submit

# Use it locally
npx i18n-rosetta sync
```

نفس الإضافة. قم بتوصيلها واختبرها.

### مجموعة الأدوات الكاملة

rosetta ليست مجرد `sync`. إنها مسار عمل متكامل للتدويل (i18n):

| الأمر | وظيفته |
|---------|-------------|
| `sync` | ترجمة المفاتيح المفقودة، والقديمة، والاحتياطية (fallback) |
| `watch` | المزامنة التلقائية عند تغير الملف المصدري |
| `lint` | فحص الكود المصدري بحثًا عن النصوص الثابتة (hardcoded) |
| `wrap` | التغليف التلقائي للنصوص الثابتة في استدعاءات `t()` |
| `audit` | سرد جميع القيم الاحتياطية `[EN]` غير المترجمة |
| `integrity` | اكتشاف تلف العناصر النائبة (placeholders)، ومشاكل الترميز، واكتمال صيغ الجمع في ICU |
| `seo` | إنشاء وسوم hreflang، وخرائط الموقع (sitemaps)، ومخطط JSON-LD |
| `status` | عرض إعدادات الأزواج اللغوية، والإضافات، ونتائج قياس الأداء |
| `provenance` | تدقيق تراخيص موارد الترجمة |
| `plugin` | تثبيت، وإزالة، وسرد إضافات الطرق |
| `fonts` | تنزيل خطوط الويب لمحولات نصوص PUA |
| `tm` | إدارة التخزين المؤقت لذاكرة الترجمة (الإحصائيات، المسح، لكل لغة) |
| `xliff` | تصدير/استيراد XLIFF 1.2 لمراجعة المترجمين المحترفين |

ثلاثة من هذه الأوامر — `lint`، `sync`، `audit` — تشكل مسار تكامل مستمر (CI) يكتشف النصوص الثابتة، ويترجمها، ويوقف عملية البناء (build) إذا كانت أي لغة غير مكتملة.

## الساحة

[لوحة متصدري الطرق](/leaderboard) هي لوحة النتائج. يتم توثيق كل إرسال ببصمة مرتبطة بتأكيد Git (Git commit)، وتعيين إصدار له بناءً على مجموعة بيانات محددة، وتقييمه بواسطة نفس أداة التقييم. يمكن لأي شخص المشاركة.

**ما الذي يمكنك إثباته؟** تقبل أداة التقييم تنسيق JSON. وتقبل الإضافات تنسيق JSON. يمكن اختبار أي طريقة تنتج JSON:

| النهج | مثال |
|----------|---------|
| **LLM موجه** | حقن القواعد النحوية والقواميس في موجه (prompt) نموذج متقدم |
| **نموذج مضبوط بدقة (Fine-tuned)** | تدريب نموذج مفتوح على نصوص متوازية — ولكن ليس على بيانات التقييم |
| **مسار محكوم بـ FST** | يقوم LLM بالتوليد → يتحقق محول الحالة المحدودة (FST) من الصرف → إعادة المحاولة |
| **نماذج متسلسلة** | النموذج أ يصيغ المسودة → النموذج ب يحرر لاحقًا → النموذج ج يقيم |
| **قاموس + LLM** | فرض مصطلحات معروفة من قاموس، وترك LLM يتعامل مع الباقي |
| **تطوري** | توليد مرشحين، وتقييمهم، وإحداث طفرة في الأفضل، ثم التكرار |
| **ترجمة جزئية** | ترجمة عينة يدويًا، وإثبات تطابق LLM الخاص بك، ثم الترجمة التلقائية للباقي |

قم بضبط النماذج بدقة. انشر خوارزميات تطورية. اختبر إجابات الطلاب في امتحانات اللغة. قم ببناء جداول بحث. اربط ثلاثة نماذج معًا. طالما أن طريقتك تنتج JSON، فإن أداة التقييم ستقيمها وسيقوم إطار العمل بتشغيلها.

:::danger القاعدة الوحيدة
**لا تتدرب على بيانات التقييم.** سيتم استبعاد الطرق التي تعرضت لمجموعة بيانات القياس. قم بالضبط الدقيق على أي شيء تريده. ولكن ليس على مجموعة الاختبار.
:::

هذه دعوة مفتوحة. إذا كنت تعمل مع لغة ذات موارد محدودة — كباحث، أو عضو في المجتمع، أو طالب، أو مجرد شخص مهتم — فقم ببناء طريقة، وشغل أداة التقييم، واحصل على أعلى نتيجة. المشكلة لم تُحل بعد. والبنية التحتية موجودة هنا.
