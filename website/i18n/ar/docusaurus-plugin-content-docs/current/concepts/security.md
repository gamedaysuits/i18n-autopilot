# الأمن والسلامة

تم تصميم Rosetta ليكون آمناً في البيئات المعادية — حيث قد تأتي بيانات اللغة (locale) من مصادر غير موثوقة، وحيث يمكن لأسماء الملفات المصممة خصيصاً تجاوز حدود الدليل، وحيث يمكن أن تحتوي مخرجات LLM على أي شيء.

## نموذج التهديد

| التهديد | ناقل الهجوم | التخفيف |
|--------|--------------|-----------|
| **تلوث النموذج الأولي (Prototype pollution)** | مفاتيح JSON مصممة خصيصاً (`__proto__`، `constructor`) | تُرفض في وقت التحليل |
| **اجتياز المسار (Path traversal)** | رموز اللغة مثل `../../etc/passwd` | يتم التحقق من كتابة الملفات في الأدلة المكونة |
| **تلف كتل التعليمات البرمجية** | يقوم LLM بالترجمة داخل حدود التعليمات البرمجية | الحماية باستخدام حراس Unicode (Unicode sentinel) |
| **المفاتيح المهلوسة** | يُرجع LLM مفاتيح لم يتم إرسالها | التحقق من صحة الاستجابة — تتم كتابة المفاتيح المقبولة فقط |
| **استهلاك الرموز (tokens) الخارج عن السيطرة** | حلقات إعادة المحاولة اللانهائية | وضع حد أقصى للميزانية عبر `maxRetries` |

## الحماية من تلوث النموذج الأولي

يتم التحقق من صحة جميع مفاتيح اللغة (locale) مقابل قائمة حظر قبل معالجتها:

- `__proto__`
- `constructor`
- `prototype`

يتم رفض أي مفتاح يطابق هذه الأنماط مع ظهور خطأ. يمنع هذا المهاجمين من استخدام ملفات لغة مصممة خصيصاً لتعديل النماذج الأولية لكائنات JavaScript.

## احتواء المسار

عند كتابة ملفات اللغة، يتحقق rosetta من بقاء مسار المخرجات ضمن الأدلة المكونة (`localesDir`، `contentDir`). يتم تعقيم رموز اللغة — لا يمكن لرمز مثل `../../secrets` الكتابة خارج الدليل المتوقع.

## حماية الكتل

أثناء ترجمة محتوى Markdown، يتم استبدال العناصر المهيكلة بعناصر نائبة (حراس Unicode) قبل إرسال النص إلى LLM:

1. **كتل التعليمات البرمجية** (المحاطة والمضمنة) → حارس (sentinel)
2. **الرموز القصيرة لـ Hugo** (`{{< >}}`، `{{% %}}`) → حارس  
3. **HTML الخام** → حارس
4. **متغيرات الاستيفاء (Interpolation variables)** (`{{ .Count }}`) → حارس

بعد الترجمة، يتم استبدال الحراس بالمحتوى الأصلي. لا يرى LLM أبداً كتل التعليمات البرمجية أو الرموز القصيرة أو HTML — لذا لا يمكنه إتلافها.

## التحقق من صحة الاستجابة

عندما يُرجع LLM استجابة JSON، يتحقق rosetta مما يلي:
- ظهور المفاتيح التي تم إرسالها في الدفعة فقط في الاستجابة
- عدم إدخال أي مفاتيح إضافية
- إمكانية تحليل الاستجابة كـ JSON صالح

يتم إسقاط المفاتيح المهلوسة بصمت. يمنع هذا مخرجات LLM من إدخال ترجمات غير متوقعة في ملفات اللغة الخاصة بك.

## بوابة الجودة

يتم التحقق من صحة كل ترجمة من خلال خمسة فحوصات حتمية قبل كتابتها على القرص. راجع [بوابة الجودة](/docs/concepts/quality-gate) للحصول على التفاصيل.

## التراجع الأسي (Exponential Backoff)

تستخدم استدعاءات API التراجع الأسي مع التذبذب (jitter) عند استلام استجابات 429 (حد المعدل) و 5xx (خطأ في الخادم). تمنع ثلاث محاولات إعادة مع زيادة التأخير الضغط الزائد على API أثناء فترات الانقطاع.

## مهلة الطلب

يحتوي كل طلب API على مهلة قدرها 30 ثانية عبر `AbortController`. يمنع هذا عملية المزامنة من التعليق إلى أجل غير مسمى بسبب اتصال ميت.

## وضع التراجع (Fallback Mode)

عندما تكون API غير متاحة، يكتب `--fallback` عناصر نائبة مسبوقة بـ `[EN]` بدلاً من الترجمات الحقيقية:

```bash
npx i18n-rosetta sync --fallback
```

```json
{
  "hero.title": "[EN] Welcome to our platform"
}
```

يتم اكتشاف هذه العناصر النائبة تلقائياً وإعادة ترجمتها في المزامنة التالية باستخدام مفتاح API صالح. لا يتم التعامل معها أبداً على أنها "مترجمة" — سيقوم `audit` بوضع علامة عليها.

## الاختبار

يتم التحقق من خصائص الأمان بواسطة مجموعة الاختبارات المعادية:

```bash
npm run test:redteam    # prototype pollution, path traversal, encoding attacks
```
