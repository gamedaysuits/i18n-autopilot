# مجموعات بيانات التقييم

مجموعات البيانات هي الأهداف الثابتة التي تعمل أداة الاختبار (harness) عليها. كل مجموعة بيانات عبارة عن ملف JSON يحتوي على أزواج من المصدر→الهدف مع مراجع ذهبية قياسية (gold-standard references). تقوم أداة الاختبار بتقييم مخرجات النموذج بناءً على هذه المراجع — ولا تقوم بتعديلها أبدًا.

:::danger لا تقم بالتدريب على بيانات التقييم

⚠️ **مجموعات البيانات هذه مخصصة للتقييم فقط.** الأساليب التي يتم تدريبها، أو ضبطها الدقيق (fine-tuned)، أو توجيهها بأمثلة قليلة (few-shot-prompted)، أو التي تتعرض بأي شكل آخر لبيانات التقييم، ستنتج درجات مضخمة بشكل مصطنع وسيتم **استبعادها من لوحة الصدارة (leaderboard).**

استخدم مدونات نصية (corpora) منفصلة للتدريب. يجب أن تظل مجموعات التقييم غير مرئية لنموذجك أثناء التطوير.
:::

## تنسيق مجموعة البيانات

تتبع كل مجموعة بيانات نفس مخطط JSON:

```json
{
  "dataset": {
    "id": "dataset-slug",
    "version": "1.0",
    "language_pair": "EN→CRK",
    "description": "Human-readable description of the dataset",
    "source_language": "en",
    "target_language": "crk",
    "created": "2025-05-01",
    "license": "CC-BY-NC-4.0",
    "provenance": ["gold_standard", "textbook"]
  },
  "entries": [
    {
      "index": 0,
      "source_text": "Hello",
      "target_expected": "tânisi",
      "difficulty": "easy",
      "provenance": "gold_standard",
      "notes": "Common greeting, SRO orthography"
    }
  ]
}
```

### كتلة `dataset` ذات المستوى الأعلى

| الحقل | النوع | الوصف |
|-------|------|-------------|
| `id` | `string` | معرّف فريد لمجموعة البيانات (يُستخدم في بطاقات التشغيل ولوحة الصدارة) |
| `version` | `string` | الإصدار الدلالي. تؤدي زيادة هذا الرقم إلى إبطال مقارنات بطاقات التشغيل السابقة |
| `language_pair` | `string` | تسمية العرض (مثل، `EN→CRK`) |
| `description` | `string` | ملخص مقروء للبشر |
| `source_language` | `string` | رمز لغة المصدر حسب BCP 47 |
| `target_language` | `string` | رمز لغة الهدف حسب BCP 47 |
| `created` | `string` | تاريخ الإنشاء حسب ISO 8601 |
| `license` | `string` | معرّف ترخيص SPDX |
| `provenance` | `string[]` | قائمة بعلامات المصدر (provenance tags) المستخدمة عبر الإدخالات |

### حقول الإدخال

| الحقل | النوع | الوصف |
|-------|------|-------------|
| `index` | `number` | فهرس إدخال يبدأ من الصفر. يجب أن يكون فريدًا ومتسلسلًا |
| `source_text` | `string` | النص المصدر المراد ترجمته |
| `target_expected` | `string` | الترجمة المرجعية الذهبية القياسية |
| `difficulty` | `string` | مستوى الصعوبة: `easy`، `medium`، `hard` |
| `provenance` | `string` | أصل هذا الإدخال (مثل، `gold_standard`، `textbook`، `elicited`) |
| `notes` | `string` | سياق اختياري للمراجعين البشريين |

## مجموعات البيانات المتاحة

### مجموعة تطوير EDTeKLA الإصدار 1

أول مجموعة بيانات للتقييم، تم إنشاؤها للترجمة من الإنجليزية إلى Plains Cree (SRO). تم إنشاؤها بواسطة [مجموعة أبحاث EdTeKLA](https://spaces.facsci.ualberta.ca/edtekla/) في جامعة ألبرتا (University of Alberta).

| الخاصية | القيمة |
|----------|-------|
| **المعرّف (ID)** | `edtekla-dev-v1` |
| **الإصدار** | `1.0` |
| **زوج اللغات** | EN → CRK (Plains Cree، قواعد الإملاء SRO) |
| **عدد الإدخالات** | 124 |
| **توزيع الصعوبة** | سهل، متوسط، صعب |
| **المصدر (Provenance)** | `gold_standard` (تم التحقق منه بواسطة متحدثين)، `textbook` (مواد تعليمية منشورة) |
| **الترخيص** | [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/) |

**ما تختبره:**

- التحيات الأساسية والعبارات الشائعة
- حيوية الاسم (Noun animacy) والإقصاء (obviation)
- تصريف الأفعال عبر الأشخاص والأزمنة
- التراكيب المكانية (Locative constructions)
- النماذج الصرفية للملكية (Possessive paradigms)
- تراكيب الجمل المعقدة

:::tip لماذا 124 إدخالًا؟
مجموعة البيانات صغيرة ومنسقة عمدًا. تم التحقق من كل إدخال بواسطة متحدثين طلقاء أو تم الحصول عليه من كتب مدرسية منشورة للغة Cree. إن مجموعة بيانات صغيرة وعالية الجودة بمعايير ذهبية تم التحقق منها هي أكثر فائدة من مجموعة كبيرة ومليئة بالضوضاء — خاصة بالنسبة للغة ذات موارد محدودة (low-resource language) حيث تكون الترجمات "القريبة بما يكفي" غالبًا غير صالحة صرفيًا.
:::

## إنشاء مجموعة بيانات جديدة

لإنشاء مجموعة بيانات لزوج لغات أو مجال جديد:

### 1. هيكلة JSON

اتبع مخطط [تنسيق مجموعة البيانات](#dataset-format). يجب أن يحتوي كل إدخال على `source_text`، و`target_expected`، و`difficulty`، و`provenance`.

### 2. تعيين معرّف فريد

استخدم معرّفًا نصيًا وصفيًا (slug): `{project}-{split}-v{version}` (مثل، `edtekla-dev-v1`، `quechua-test-v1`).

### 3. التحقق من المعايير الذهبية

يجب التحقق من كل قيمة `target_expected` بواسطة متحدث طليق أو الحصول عليها من مصدر منشور وخاضع لمراجعة الأقران. المراجع المُنشأة آليًا تُبطل الغرض من التقييم.

### 4. تحديد مستويات الصعوبة

قم بتعيين مستوى صعوبة لكل إدخال:

| المستوى | المعايير |
|------|----------|
| `easy` | عبارات قصيرة، مفردات شائعة، بنية صرفية بسيطة |
| `medium` | جمل كاملة، تعقيد صرفي متوسط |
| `hard` | قواعد نحوية معقدة، تراكيب نادرة، محتوى خاص بثقافة معينة |

### 5. وضع علامات المصدر (Provenance)

يجب أن يشير كل إدخال إلى مصدره. العلامات الشائعة هي:

- `gold_standard` — تم التحقق منه بواسطة متحدثين طلقاء
- `textbook` — من مواد تعليمية منشورة
- `elicited` — تم إنتاجه من خلال جلسات استنباط منظمة
- `corpus` — مستخرج من مدونة نصية متوازية (parallel corpus)

### 6. التحقق من صحة الملف

قم بتشغيل أداة الاختبار (harness) على مجموعة بياناتك باستخدام أي نموذج للتحقق من أن ملف JSON مصاغ بشكل صحيح وأن جميع الحقول المطلوبة موجودة:

```bash
python eval/baseline_experiment.py --dataset path/to/your-dataset.json
```

ستصدر أداة الاختبار خطأً عند وجود حقول مفقودة، أو فهارس مكررة، أو انتهاكات للمخطط.

### 7. الإرسال للإدراج

افتح طلب سحب (pull request) في [مستودع أداة التقييم (eval harness repository)](https://github.com/gamedaysuits/gds-mt-eval-harness) مع وضع ملف مجموعة بياناتك في الدليل `data/`. قم بتضمين توثيق لمنهجية التحقق الخاصة بك ومصادر المنشأ.

## FLORES+ Devtest

معيار قياس متعدد اللغات واسع التغطية تديره [مبادرة بيانات اللغة المفتوحة (OLDI)](https://huggingface.co/datasets/openlanguagedata/flores_plus). يُستخدم لمعيار الحدود متعدد النماذج الخاص بـ rosetta.

| الخاصية | القيمة |
|----------|-------|
| **المعرّف (ID)** | `flores-plus-devtest` |
| **أزواج اللغات** | EN → 39 لغة (جميع اللغات الطبيعية المسجلة في rosetta) |
| **عدد الإدخالات** | 1,012 جملة لكل لغة |
| **الترخيص** | [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/) |
| **المصدر** | في الأصل Meta FLORES-200، وتتم إدارته الآن بواسطة OLDI |
| **الموقع** | بيانات اختبار ثابتة (fixtures) مستخرجة مسبقًا في `test/benchmark/fixtures/` في مستودع rosetta الرئيسي |

:::danger للتقييم فقط
مجموعة FLORES+ مخصصة للتقييم فقط. يطلب القيمون عليها صراحةً **عدم استخدامها كبيانات تدريب**. تأكد من استبعاد محتوياتها من أي مدونات تدريبية (training corpora).
:::
